A Software Engineering Methodology: DevOps

DevOps is a software engineering methodology which aims at removing differences between Development and Operational side. It is a strategy used throughout the application release process which forms an infinite loop between both the sides.

Let's See!

Stages

Various stages like Version Control (Source Code Management), Continuous Building (Continuous Integration), Continuous Testing and Delivering, Continuous Deployment involving Configuration Management and Containerization, then finally managed and monitored.

Learn More
Version Control

Contains different versions of the code, done by GitHub or subversion.

Continuous Building

Contains planning, building, compiling, code reviewing, etc.

Continuous Integration

Contains tasks like validating, unit testing, integration testing, etc and is done by Jenkins.

Continuous Delivery

Deploying build application to test servers (automation testing).

Continuous Deployment

Deploying tested application on prod server for release.

Management

Live application monitored by Splunk, Nagios, etc.

Used Tools

Git/GitHub

GitHub manages the source code and its versions. Git is a distributed management tool which maintains the remote and local repositories.

Jenkins: The heart of DevOps

It integrates the developing and operating sections. If a change in source code is found, it redirects to planning stage.

Docker: For Containerization

It creates lightweight containers as an alternate to virtual machines, stores binaries and libraries for the application.

Issues Resolved by DevOps

How does DevOps add value to customers?

It makes use of innovative technologies and reduce task redundancy, hence faster delivery. Its ability to roll out new feautures as per customer requirements, makes it effective.

How does DevOps reduce production cost?

Due to constant testing, errors get fixed quickly, hence it speeds up time to market. This makes teams move faster and the ability to scale also increases.

How does DevOps ensure high quality?

Quick feedbacks given within teams reduces response time to customers, hence requirements are met and new feaures are rolled out.

How does DevOps reduce labour?

It eliminates manual work, overhead of launching and updating is also reduced, hence overall product development cost is optimized.

How does DevOps reduce complexity?

It speeds up tasks and reduces cycle time. Teams move faster and incorporate feedbacks with cooperation reducing development complexity.

Which companies have adopted DevOps?

In the recent times, some of the major companies like Amazon, Walmart, Target, etc have adopted this methodology.

Adopted by Companies

Amazon
E-Commerce

Amazon started using Amazon Web Services (AWS), and maintaining changes and updations of code using tools like Git and GitHub. This helped Amazon in maintaining and monitoring code of the software.

Netflix
Media Service

Netflix started using its self-developed container tool called 'Titus' to maintain changes and hence handle user traffic and heavy data in a single platform.

HP
Product-Based

To analyse development of all the models on a single platform, continuous integration and continuous deployment (CI/CD) was adopted. This reduced errors and redundancies in the code.

Facebook
Social Media

Facebook shifted to configuration management which initiated quick user updates and data recovery as compared to previous times. The entire code is now stored at one place.
